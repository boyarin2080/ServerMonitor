# Инструкция по запуску Server Monitor в Docker на Linux

## Предварительные требования
- Установленный Docker на Linux системе
- Доступ к терминалу с правами sudo или добавление пользователя в группу docker

## Сборка и запуск контейнера

### 1. Сборка Docker образа
Перейдите в директорию с проектом и выполните команду сборки:

```bash
docker build -t server-monitor .
```

### 2. Запуск контейнера

#### Вариант A: Базовый запуск
```bash
docker run -d -p 5000:5000 --name server-monitor server-monitor
```

#### Вариант B: Запуск с доступом к системным метрикам хоста
Для корректной работы psutil и получения температуры CPU, контейнеру нужен доступ к системным файлам:

```bash
docker run -d \
  -p 5000:5000 \
  --name server-monitor \
  --pid=host \
  --privileged \
  server-monitor
```

**Важные моменты:**
- Флаг `--pid=host` позволяет контейнеру видеть процессы хоста
- Флаг `--privileged` дает доступ к системным датчикам (включая температуру)
- Это нужно для корректной работы `psutil.sensors_temperatures()`

### 3. Проверка работы приложения
После запуска откройте браузер и перейдите по адресу:
```
http://localhost:5000
```

Для проверки API:
```
http://localhost:5000/api/stats
```

### 4. Управление контейнером

#### Просмотр логов
```bash
docker logs server-monitor
```

#### Остановка контейнера
```bash
docker stop server-monitor
```

#### Запуск остановленного контейнера
```bash
docker start server-monitor
```

#### Удаление контейнера
```bash
docker rm -f server-monitor
```

#### Удаление образа
```bash
docker rmi server-monitor
```

## Решение проблем

### Проблема: Температура CPU показывает "N/A"
Решение: Запустите контейнер с флагами `--pid=host` и `--privileged`

### Проблема: Приложение не запускается
Решение: Проверьте логи контейнера:
```bash
docker logs server-monitor
```

### Проблема: Порт 5000 уже занят
Решение: Используйте другой порт при запуске:
```bash
docker run -d -p 8080:5000 --name server-monitor server-monitor
```
Тогда приложение будет доступно по `http://localhost:8080`

## Оптимизация для production

Для production использования рекомендуется:
1. Использовать Gunicorn вместо встроенного Flask сервера
2. Добавить healthcheck в Dockerfile
3. Настроить логирование
4. Использовать Docker Compose для управления

- `servmon.py` - основной код приложения
- `requirements.txt` - зависимости Python
- `Dockerfile` - конфигурация Docker образа
